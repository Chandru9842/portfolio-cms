## 🎨 Portfolio Project Case Study

If you display this system inside your own portfolio, use this structured case study:

### **Case Study: SaaS Portfolio Content Management System**

#### **The Challenge**
As developers, our portfolios quickly become outdated. Editing source code, updating JSON files, redeploying static hosting pages, and managing multiple copies of CV files is a slow, manual process. I set out to build a platform that allows developers to manage their professional identity dynamically via a secure, production-grade interface.

#### **The Solution**
I designed a decoupled full-stack architecture that splits public, speed-optimized presentation views from write-intensive administrative management tools. Using a robust **Java / Spring Boot** core, I established secure endpoints guarded by **Spring Security** and **JWT access keys**. On the front end, **React 19** and **TypeScript** power a responsive, modern glassmorphic Bento-Grid UI. Content updates in the admin dashboard reflect on the public site instantly without requiring a code rebuild or redeployment.

#### **Key Engineering Milestones**
*   **Decoupled State Syncing**: Developed custom state controllers using React Context to keep the interactive editor preview synchronized with active server data.
*   **Performance Optimization**: Implemented reverse proxies via Vercel CDN routing which decreased CORS preflight overhead and increased client-side Core Web Vital scores.
*   **Transactional Integrity**: Configured JPA transactional annotations across the Spring Boot service layer to ensure atomic database rollbacks during multi-step updates.
*   **Security Auditing**: Implemented standard authentication filters that capture client browser and geolocation attributes, populating a security log dashboard for unauthorized login detection.
